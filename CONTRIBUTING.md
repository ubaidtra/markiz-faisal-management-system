# Contributing to Faisal Center Management System

Thank you for your interest in contributing to the Faisal Center Management System!

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/fisal-center-management-system.git`
3. Install dependencies: `npm run install-all`
4. Set up environment variables (see README.md)
5. Start development servers: `npm run dev`

## Code Style

- Use consistent indentation (2 spaces)
- Follow existing code patterns
- Add comments for complex logic
- Use meaningful variable and function names

## Commit Messages

Use clear, descriptive commit messages:
- `feat: Add new feature`
- `fix: Fix bug in student registration`
- `docs: Update README`
- `style: Format code`
- `refactor: Refactor authentication logic`

## Pull Request Process

1. Create a feature branch: `git checkout -b feature/your-feature-name`
2. Make your changes
3. Test thoroughly
4. Commit your changes: `git commit -m "feat: Add your feature"`
5. Push to your fork: `git push origin feature/your-feature-name`
6. Create a Pull Request

## Testing

- Test all new features before submitting
- Ensure existing functionality still works
- Test on different browsers and devices
- Check mobile responsiveness

## Reporting Issues

When reporting issues, please include:
- Description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Browser/OS information

## Questions?

For questions or support, please open an issue on GitHub.

Thank you for contributing!
